ServerManager Framework

Welcome to the ServerManager documentation. This framework is a custom-built, Express-like HTTP server created entirely from scratch using Node.js's native net module. It simplifies the process of handling raw TCP connections and HTTP protocols, allowing developers to focus on building robust APIs.

Core Features

  • Express-style Routing: Easily define GET and POST routes.
  • Request Parsing: Automatically parses query parameters and extracts JSON bodies from POST requests.
  • Custom Response Objects: Chainable response methods like res.status().json() or res.send().
  • Static File Serving: Built-in support for serving static assets (HTML, CSS, JS, images).

The 'req' and 'res' Objects

The core of ServerManager's developer experience relies on two intelligent objects passed to every route handler.

1. The Request Object (req)

Instead of manually parsing raw HTTP strings, the req object organizes client data:

  • req.method - The HTTP method used (e.g., 'GET').
  • req.path - The clean URL path requested (e.g., '/api/users').
  • req.query - A dictionary containing URL parameters (e.g., ?name=Roee becomes { name: 'Roee' }).
  • req.body - For POST requests, contains the parsed payload sent by the client.

2. The Response Object (res)

The res object abstracts the complexity of building valid HTTP responses over raw sockets. It supports method chaining for elegant code syntax.

  • res.status(code) - Sets the HTTP status code (e.g., 200, 404). Returns this to allow chaining.
  • res.send(text) - Sends a plain text response and closes the connection.
  • res.json(data) - Serializes a JavaScript object into a JSON string, sets the correct headers, and sends it to the client.
  • res.socket - Provides access to the underlying TCP socket, necessary for tasks like serving static files.

Usage Examples

Initialization & Setup

const ServerManager = require('./ServerManager');
const app = new ServerManager();

Basic GET Route

Handling a simple GET request and sending plain text.

app.get('/ping', (req, res) => {
    res.status(200).send("Pong! Server is alive.");
});

Working with Query Parameters

Extracting data from the URL query string and returning JSON.

// Request: GET /api/greeting?name=Alice
app.get('/api/greeting', (req, res) => {
    const user = req.query.name || "Guest";
    
    res.status(200).json({
        success: true,
        message: `Hello there, ${user}!`
    });
});

Handling POST Requests and JSON Body

Receiving JSON data from the client, parsing it, and responding appropriately.

// Request: POST /api/login
// Body: { "username": "admin", "password": "123" }
app.post('/api/login', (req, res) => {
    const payload = req.body;

    if (!payload || !payload.username) {
        return res.status(400).json({ error: "Username is required" });
    }

    res.status(200).json({
        success: true,
        message: `Welcome back, ${payload.username}`
    });
});

Serving Static Files

Delivering an HTML file to the browser. Note the use of res.socket.

app.get('/', (req, res) => {
    app.serveStaticFile(res.socket, './public/index.html');
});

Starting the Server

app.listen(3000);
console.log("Server listening on port 3000");
